Redux 是專門管理 state 的函式庫 單一來源原則：整個應用程式的狀態儲存在一個單一的 JavaScript 物件中。 狀態是唯讀的：唯一改變狀態的方法是發送（dispatch）一個行動（action）。 純函數用於處理狀態變化：行動被傳遞給純函數（稱為 reducer）以更新狀態。 狀態存儲在一個單一的狀態樹中，然後通過行為（actions）和純函數（reducers）來更新這些狀態 狀態管理變得更加可預測、可追蹤和易於除錯。 純函數（稱為 reducers）-可預測: 這意味著對於相同的輸入，reducers 始終返回相同的輸出，這讓狀態更新變得可預測。 行動（action）來觸發-可追蹤: 這使得開發者能夠輕易地追蹤到狀態何時、如何改變。
